# Home-first, not chat-first

The founding pain is "being forced to live inside a chat, no clear view at first glance". A chat with GenUI in its responses is still a stream (and it already exists: OpenClaw A2UI Canvas, Gemini Dynamic View). We decide that the primary interface is a **Home of persistent Surfaces per life area**, maintained by the Agent; chat is an ever-present input that *modifies the Surfaces*. Verified as open territory as of mid-2026 (the closest competitor, Skye, has not launched and is tied to WidgetKit iOS) — see `docs/references/02-competitor-home-first.md`.

Status: accepted

## Considered Options
- Chat-first enriched with interactive components: rejected — territory already occupied, and "at first glance" you would still see a chat.
